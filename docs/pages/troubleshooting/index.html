Most Common Issues

Python Version Error

HIGH

Error message: "Python 3.8+ required" when trying to run framework tools or setup wizard.

Solution Steps

  1. Check your current Python version: python --version
  2. Download Python 3.8+ from python.org
  3. Update your system PATH to include Python
  4. Verify installation: python3 --version
  5. Reinstall framework dependencies

Permission Denied

MEDIUM

Getting permission errors when trying to install packages or create directories during setup.

Solution Steps

  1. Try using pip install --user for user-level installation
  2. Or use sudo pip install (not recommended)
  3. Create virtual environment: python -m venv myenv
  4. Activate environment and install within it
  5. Check directory permissions for project folder

Missing AI_RULES.md

HIGH

AI assistant reports missing AI_RULES.md file or cannot find project-specific rules.

Solution Steps

  1. Run validation: python tools/validator.py --check-config
  2. Check if AI_RULES.md exists in project root
  3. Copy template: cp templates/core-files/AI_RULES.md .
  4. Customize rules for your specific project needs
  5. Validate again to ensure proper configuration

Task Execution Fails

HIGH

AI assistant cannot execute tasks properly, gets stuck, or reports errors during task completion.

Solution Steps

  1. Verify START_TASK.md exists and is readable
  2. Check AI_RULES.md for proper task execution rules
  3. Run system validation: python tools/validator.py --check-system
  4. Check for open issues in issues/ directory
  5. Ensure AI assistant follows the protocol correctly
  6. Try manual execution if automated fails

AI Cannot Find Documentation

MEDIUM

AI assistant reports difficulty finding relevant documentation or navigation issues.

Solution Steps

  1. Verify AI_APP_GUIDE.md exists and is up to date
  2. Check documentation structure is complete
  3. Run documentation validation: python tools/validator.py --check-docs
  4. Ensure docs/ directory structure follows framework standards
  5. Update navigation guides if structure changed
  6. Test documentation access manually

AI Not Following Rules

MEDIUM

AI assistant is not following the rules specified in AI_RULES.md or ignoring project constraints.

Solution Steps

  1. Ensure AI_RULES.md is in the correct location
  2. Check that rules are clearly written and specific
  3. Verify AI assistant reads the rules file first
  4. Add more specific constraints if rules are too vague
  5. Test with simple tasks to verify rule compliance
  6. Consider adding examples to clarify expectations

File System Permissions

HIGH

Framework cannot create, modify, or delete files due to insufficient permissions.

Solution Steps

  1. Check current directory permissions: ls -la
  2. Change ownership: chown -R $USER:$USER .
  3. Set proper permissions: chmod -R 755 .
  4. Ensure write permissions on project directory
  5. Check if files are locked by another process
  6. Try running with elevated privileges if necessary

Error History Not Working

MEDIUM

Error management system is not properly tracking or retrieving historical errors.

Solution Steps

  1. Verify error-management/ directory exists
  2. Check MASTER_ERROR_INDEX.md is present and valid
  3. Validate error category directories exist
  4. Run error system check: python tools/validator.py --check-errors
  5. Ensure error templates are properly formatted
  6. Test error documentation creation manually

Prevention & Best Practices

System Integrity

Maintain system integrity by following framework conventions and using validation tools regularly.

  • Run validation checks after major changes
  • Keep AI_RULES.md updated with current requirements
  • Use version control for all documentation changes
  • Regularly backup important configurations

AI Integration

Ensure smooth AI integration by maintaining clear communication protocols and rule definitions.

  • Keep AI_RULES.md clear and specific
  • Test AI commands before complex tasks
  • Maintain consistent task initiation patterns
  • Document successful AI interaction patterns

Documentation Management

Keep documentation current and accessible through regular maintenance and organization.

  • Update documentation after code changes
  • Maintain consistent documentation structure
  • Keep AI_APP_GUIDE.md current with navigation
  • Regularly validate documentation links

Issue Prevention

Prevent issues through proactive monitoring and systematic problem resolution.

  • Address incomplete tasks promptly
  • Monitor error patterns for systemic issues
  • Keep issue tracking system organized
  • Learn from past errors to prevent recurrence
