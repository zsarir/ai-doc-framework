📍 Critical File Placement

The most important aspect of the framework - where files belong

🚨 CRITICAL: ai-doc-config.json Placement

This file is the heart of the entire framework and MUST be placed in your project root directory.

✅ CORRECT Structure

Project Root Structure 
your-project/                     # 📍 PROJECT ROOT
├── 🎯 ai-doc-config.json         # 🏛️ CENTRAL CONTROL FILE
├── 📄 AI_RULES.md               # AI behavior rules
├── 📄 START_TASK.md             # Task initialization
├── 📄 COMPLETE_TASK.md          # Task completion
├── 📁 docs/                     # Project documentation
├── 📁 error-management/         # Error system
├── 📁 issues/                   # Issue tracking
└── 📁 [applications]/          # Your applications

❌ INCORRECT Placement

Wrong Locations 
your-project/
├── 📁 docs/
│   └── ai-doc-config.json       # ❌ WRONG - Inside docs
└── 📁 src/
    └── ai-doc-config.json       # ❌ WRONG - Inside src

🔍 How to Verify Correct Placement

Verification Commands 
# Check if file exists in project root
ls -la ai-doc-config.json

# Verify location (should be in current directory)
find . -name "ai-doc-config.json" -type f

# Should return: ./ai-doc-config.json
# NOT: ./docs/ai-doc-config.json

🔄 System Flow and Data Flow

Understanding how the framework processes tasks and manages information

📊 Task Execution Flow

AI Task Processing Flow 
1. 🎯 User Request → AI receives task
2. 📋 AI_RULES.md → AI reads behavior rules
3. 🎯 ai-doc-config.json → AI gets project structure
4. 📄 START_TASK.md → AI follows initialization protocol
5. 📚 docs/AI_APP_GUIDE.md → AI navigates documentation
6. 🚨 error-management/ → AI checks for similar errors
7. 📋 issues/ → AI checks for open issues
8. 🔧 Execute Task → AI performs the work
9. ✅ COMPLETE_TASK.md → AI follows completion protocol

🛡️ System Integrity and Validation

How the framework ensures system reliability and prevents corruption

🔍 Integrity Checks

📁 File Existence Validation

File Validation Logic 
def validate_critical_files():
    critical_files = [
        "./ai-doc-config.json",
        "./AI_RULES.md",
        "./START_TASK.md",
        "./COMPLETE_TASK.md"
    ]

    for file_path in critical_files:
        if not exists(file_path):
            alert_user(f"Missing critical file: {file_path}")
            return False
    return True

⚙️ Configuration Validation

Config Validation Logic 
def validate_ai_doc_config():
    required_sections = [
        "project", "applications", "error_categories", "issue_categories"
    ]

    config = load_json("./ai-doc-config.json")

    for section in required_sections:
        if section not in config:
            alert_user(f"Missing section in config: {section}")
            return False
    return True

🏗️ Structure Validation

Structure Validation Logic 
def validate_project_structure():
    required_dirs = [
        "./docs", "./error-management", "./issues"
    ]

    for dir_path in required_dirs:
        if not exists(dir_path):
            alert_user(f"Missing directory: {dir_path}")
            return False
    return True

🔧 Troubleshooting Common Issues

Solutions to the most common problems users encounter

❓ "ai-doc-config.json not found"

🔍 Symptoms:

  • Setup wizard fails
  • Validation errors occur
  • System cannot initialize

✅ Solutions:

Fix Commands 
# Check if file exists in project root
ls -la ai-doc-config.json

# Verify location
find . -name "ai-doc-config.json" -type f

# Re-run setup wizard
python tools/setup-wizard.py

❓ "File already exists" errors

🔍 Symptoms:

  • Cannot create new files
  • Duplicate file warnings
  • System integrity errors

✅ Solutions:

Resolution Steps 
# Check existing files
find . -name "*.md" -type f

# Use existing files instead
# Check AI_RULES.md for rules
cat AI_RULES.md | grep -A 10 "File Creation"

❓ Configuration validation fails

🔍 Symptoms:

  • Config file format errors
  • Missing required sections
  • Invalid JSON structure

✅ Solutions:

Validation Commands 
# Validate JSON syntax
python -c "import json; json.load(open('ai-doc-config.json'))"

# Check required sections
python tools/validator.py --check-config

❓ Performance issues

🔍 Symptoms:

  • Slow file loading
  • High memory usage
  • Long validation times

✅ Solutions:

Performance Fixes 
# Enable caching
export AI_DOC_FRAMEWORK_CACHE=1

# Optimize configuration
python tools/optimizer.py

# Monitor performance
python tools/performance-monitor.py
