Usage Guide

Learn how to effectively use the AI Documentation Framework to optimize your development workflow and documentation management.

Basic Workflow

The framework follows a simple but powerful workflow that AI assistants can follow automatically.

Framework Workflow Overview

1

Setup

Initialize the framework for your project using the setup wizard or manual configuration.

2

Task Execution

Use START_TASK.md for AI task initiation and automatic navigation through documentation.

3

Documentation

AI automatically navigates and updates documentation based on task requirements.

4

Error Management

Errors are automatically documented and searchable for future reference.

5

Issue Tracking

Incomplete tasks and unresolved errors are tracked for systematic resolution.

6

Completion

Use COMPLETE_TASK.md for task finalization and cleanup.

AI Navigation Guide

How AI assistants automatically navigate documentation based on task requirements.

Single Application Tasks

  1. Read web/START_TASK.md
  2. Validate web application system integrity
  3. Check for open web-specific issues
  4. Read web/AI_RULES.md for web-specific rules
  5. Navigate web documentation using web/docs/AI_APP_GUIDE.md
  6. Search web error history
  7. Execute task
  8. Use web/COMPLETE_TASK.md for completion

Multi-Application Tasks

  1. Read root START_TASK.md
  2. Validate entire system integrity
  3. Check for open issues across all applications
  4. Read root AI_RULES.md for project-wide rules
  5. Navigate project documentation using docs/AI_APP_GUIDE.md
  6. Search error history across all applications
  7. Coordinate task execution across applications
  8. Use root COMPLETE_TASK.md for completion

Error Investigation

  1. Search existing errors in error-management/MASTER_ERROR_INDEX.md
  2. Check category-specific errors in error-management/[category]/ERROR_INDEX.md
  3. Document new errors using the error template
  4. Update error indices with new solutions
  5. Apply known solutions if available
  6. Document resolution process
  7. Update related documentation

Documentation Updates

  1. Identify documentation requiring updates
  2. Read current documentation structure
  3. Update relevant sections based on changes
  4. Validate documentation accuracy
  5. Update AI_APP_GUIDE.md navigation
  6. Update VERSION.md with changes
  7. Refresh error documentation if needed

Best Practices

Guidelines for getting the most out of the AI Documentation Framework.

For AI Assistants

  • Always start with START_TASK.md
    Never skip the initialization protocol - it ensures proper system setup and integrity validation.
  • Search error history first
    Don't repeat known solutions - always check existing error documentation before implementing fixes.
  • Document everything
    Keep the knowledge base updated with new solutions, errors, and patterns discovered during tasks.
  • Use the cleanup system
    Always run cleanup automation to remove temporary files and debug code after task completion.

For Developers

  • Customize AI_RULES.md
    Add project-specific rules, constraints, and requirements to guide AI behavior.
  • Keep documentation updated
    Regularly review and update documentation to maintain accuracy and relevance.
  • Review issue tracking
    Regularly check and resolve issues logged by the AI assistant to maintain system health.
  • Monitor effectiveness
    Track framework performance metrics and adjust configuration as needed.
